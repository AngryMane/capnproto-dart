MessageReaderOptions

MessageReaderOptions class

Configures resource limits applied while decoding a message.

Intended users

  • Application and library developers working with Cap'n Proto messages.

Primary use cases

  • Reads, writes, validates, or reports failures for application messages.

maxSegments → int
Maximum number of segments a message's framing header may declare. Guards against a small message claiming an enormous segment count and forcing a correspondingly large up-front allocation before any content is read. Legitimate messages essentially never need more than a handful of segments. Default: 512.
final
